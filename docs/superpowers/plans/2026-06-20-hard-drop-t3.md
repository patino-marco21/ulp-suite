# Hard-drop T3 Credentials Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep T1, T2, and country-unknown credentials while permanently rejecting new T3 rows and safely deleting the existing T3 backlog.

**Architecture:** Extend the existing ingest policy with a non-overridable hard-tier set evaluated before keep suffixes. Forward a default `T3` hard-drop through Compose, and add a dedicated fixed-predicate ClickHouse purge script that is dry-run-first, polls mutations, and verifies zero remaining T3 rows.

**Tech Stack:** TypeScript, Vitest, Next.js 15, Docker Compose, ClickHouse 26.x, Bash.

## Global Constraints

- Preserve all T1, T2, and country-unknown rows.
- Hard-drop T3 before keep-suffix evaluation; `.sa` cannot override the drop.
- Do not combine regional deletion with malformed-row cleanup.
- Never print credential passwords from operational scripts.
- Production purge requires a verified off-host backup.
- Shell scripts committed by this plan use LF line endings.

function parseTierSet(raw: string | undefined): Set<string> {
  return new Set(
    (raw ?? '')
      .split(',')
      .map(t => t.trim().toUpperCase())
      .filter(t => VALID.has(t)),
  )
}

- [ ] **Step 2: Implement the purge script**

The script must use `set -euo pipefail`, verify the Compose project/container, print counts and a password-free sample, and exit after dry run unless `APPLY=1`.

Submit:

```sql
ALTER TABLE ulp.credentials
DELETE WHERE country_tier = 'T3'
SETTINGS mutations_sync = 0
```

Capture the newest mutation id for the table, poll `system.mutations` for `is_done`, `parts_to_do`, and `latest_fail_reason`, fail immediately on a nonempty failure reason, and fail after `TIMEOUT_SECONDS`. After completion, run:

```sql
SELECT countIf(country_tier = 'T3') AS remaining_t3
FROM ulp.credentials
```

Exit nonzero unless the result is exactly `0`.
